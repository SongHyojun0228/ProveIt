# claude.md — ProveIt: 과정 추적형 포트폴리오 플랫폼

## 1. 서비스 정체성

**ProveIt**은 "이 사람이 실제로 어떻게 일하는지"를 자동으로 기록하고, 검증 가능한 포트폴리오로 만들어주는 플랫폼이다. GitHub가 개발자의 코드를 추적하듯, ProveIt은 모든 분야의 작업 과정을 추적한다.

핵심 가치 제안: **"결과물이 아니라 과정을 증명한다."**

학위는 "이 과정을 통과했다"를 말하고, 이력서는 "이런 일을 했다"를 말한다. ProveIt은 **"이렇게 사고하고 이렇게 만들었다"**를 말한다. AI 시대에 이것만이 위조가 어렵다.

## 2. 핵심 AI 에이전트 구조

### 2.1 Process Tracker Agent (핵심 엔진)

**역할:** 사용자의 작업 과정을 백그라운드에서 자동 기록

**입력 소스:**
- IDE 활동 (VS Code, Cursor, JetBrains 확장)
  - 파일 생성/수정/삭제 타임라인
  - 코드 변경 diff (라인 단위)
  - AI 코파일럿 호출 여부와 수락/거부/수정 비율
  - 터미널 명령어 패턴 (빌드, 테스트, 디버그 순서)
  - 에러 발생 → 해결까지의 시간과 시도 횟수
- 디자인 도구 (Figma 플러그인, 향후)
  - 프레임 생성/수정 히스토리
  - 컴포넌트 반복 횟수
  - 피드백 코멘트 반영 추적
  - Shadcn
- 문서 도구 (Notion, Google Docs API, 향후)
  - 문서 버전 히스토리
  - 리서치 노트 → 초안 → 최종본 전환 과정
  - 외부 소스 참조 패턴

**처리 로직:**
```
원시 활동 로그
  → 노이즈 필터링 (커피 브레이크, 무관한 탭 전환 제거)
  → 세션 분할 (의미 있는 작업 단위로 묶기)
  → 의사결정 포인트 추출 ("여기서 A 대신 B를 선택했다")
  → AI 활용 비율 계산 (AI 생성 코드 vs 직접 작성 코드)
  → 타임스탬프 서명 (변조 방지)
```

**출력:** 구조화된 작업 세션 데이터 (JSON)

### 2.2 Insight Synthesizer Agent

**역할:** 원시 기록을 인간이 읽을 수 있는 포트폴리오 스토리로 변환

**하는 일:**
- 프로젝트별 요약 생성
  - "40시간에 걸쳐 완성. AI 도구 활용률 35%. 핵심 아키텍처 결정 3건은 직접 판단."
  - "가장 오래 걸린 문제: 인증 로직 (12시간, 7가지 접근 시도 후 해결)"
  - "주요 강점: 디버깅 시 체계적 접근 (가설→검증→반복 패턴 일관됨)"
- 역량 태그 자동 추출
  - 기술 스택 (사용 빈도, 숙련도 수준)
  - 문제 해결 패턴 (시행착오형 vs 체계적 분석형 vs 문서 참조형)
  - 협업 패턴 (PR 리뷰 참여도, 코멘트 품질)
- 시간 경과에 따른 성장 곡선
  - "3개월 전 대비 디버깅 속도 40% 향상"
  - "새로운 프레임워크 학습 시 평균 2주 내 생산적 수준 도달"

**프롬프트 설계 원칙:**
- 과장하지 않는다. 데이터에 기반한 팩트만 서술
- "이 사람이 좋다"가 아니라 "이 사람은 이렇게 일한다"를 보여준다
- 약점도 건설적으로 표현한다 ("테스트 작성 비율이 평균 이하 → 개선 추세")

### 2.3 Verification Agent

**역할:** 포트폴리오의 진위 검증

**검증 메커니즘:**
- 타임스탬프 무결성 검증 (서버 시간 + 클라이언트 시간 교차 확인)
- 활동 패턴 일관성 검사 (갑자기 코딩 스타일이 바뀌면 플래그)
- AI 생성물 vs 인간 작성물 분류 (키스트로크 패턴, 수정 빈도, 사고 흔적 유무)
- 옵션: 행동 바이오메트릭스 (타이핑 리듬, 마우스 패턴 — 사용자 동의 하에)

**신뢰 등급:**
- 🟢 High Confidence: 지속적 활동 기록 + 일관된 패턴 + 타임스탬프 무결
- 🟡 Medium: 간헐적 기록 또는 일부 불일치
- 🔴 Low: 대량의 즉시 생성 코드, 패턴 불일치

### 2.4 Employer View Agent

**역할:** 채용 담당자가 조회할 때 최적화된 뷰 제공

**하는 일:**
- JD(직무기술서) 입력 → 해당 역량에 관련된 프로젝트/활동만 필터링
- "이 지원자는 React 프로젝트에서 평균 대비 30% 빠르게 컴포넌트를 구축합니다"
- "AI를 도구로 효과적으로 활용하되 핵심 설계 판단은 직접 내리는 유형입니다"
- 다른 지원자와의 익명 비교 (옵트인한 사용자 간)

## 3. 데이터 아키텍처

### 3.1 데이터 흐름

```
[사용자 로컬 환경]
  IDE 확장 / 브라우저 확장 / 데스크톱 앱
    ↓ (암호화된 활동 로그)
[ProveIt 서버]
  Process Tracker Agent → 원시 데이터 저장
  Insight Synthesizer → 포트폴리오 생성
  Verification Agent → 신뢰 등급 부여
    ↓
[ProveIt 대시보드]
  사용자 뷰: 내 포트폴리오 관리, 공유 설정
  고용주 뷰: 지원자 포트폴리오 조회
```

### 3.2 프라이버시 원칙 (최우선)

- **로컬 퍼스트:** 원시 데이터는 사용자 기기에 저장. 서버에는 분석된 요약만 전송 (옵트인)
- **선택적 공유:** 프로젝트별로 공개/비공개 설정 가능. 회사 프로젝트는 비공개로 두되 "이런 유형의 작업을 했다"는 메타데이터만 공유 가능
- **삭제 권한:** 사용자가 언제든 데이터 완전 삭제 가능
- **투명성:** "어떤 데이터가 수집되고 있는지" 실시간 대시보드로 보여줌

이것이 "감시 도구"가 아닌 "자기 증명 도구"가 되려면 **사용자의 통제권이 절대적**이어야 한다.

## 4. 기술 스택 (MVP)

### Phase 1: VS Code 확장 (0-3개월)
- **프론트:** VS Code Extension API (TypeScript)
- **백엔드:** Node.js + PostgreSQL
- **AI:** Claude API (Insight Synthesizer용)
- **인증:** 타임스탬프 서명 (SHA-256 해시 체인)
- **대시보드:** React + Next.js

### Phase 2: 웹 플랫폼 (3-6개월)
- Figma 플러그인 (Figma Plugin API)
- Chrome 확장 (문서 작업 추적)
- 고용주 대시보드 + ATS 연동 API

### Phase 3: 생태계 (6-12개월)
- LinkedIn 연동
- 대학 LMS 연동 (Canvas, Blackboard)
- 모바일 앱 (학습 과정 기록)

## 6. MVP 정의 (8주 스프린트)

### Week 1-2: VS Code 확장 코어
- git diff 기반 코드 변경 추적
- AI 코파일럿(Copilot/Claude) 사용 감지 (확장 API 호출 모니터링)
- 세션별 활동 로그 로컬 저장

### Week 3-4: 인사이트 엔진
- Claude API 연동 → 세션 로그를 자연어 요약으로 변환
- 프로젝트별 AI 활용 비율, 작업 시간, 주요 의사결정 포인트 추출
- 기본 대시보드 (웹)

### Week 5-6: 공유 + 검증
- 고유 URL 생성 (포트폴리오 공유 링크)
- 타임스탬프 해시 체인 (기본 변조 방지)
- 신뢰 등급 표시

### Week 7-8: 사용자 테스트
- 개발자 10명에게 2주간 사용 테스트
- 채용 담당자 5명에게 결과물 피드백
- 핵심 지표: "이 포트폴리오가 채용 판단에 도움이 되나?" (Y/N + 이유)

### MVP 성공 기준
- 사용자의 70% 이상이 "계속 사용하겠다"
- 채용 담당자의 60% 이상이 "채용에 도움이 된다"
- 일평균 활성 기록 시간 2시간 이상

## 7. 경쟁 포지셔닝

```
                    과정 추적 깊이
                        ↑
                        |
              ProveIt ★ |
                        |
            GitHub ●    |    (빈 공간)
                        |
    Credly ●            |          ● LinkedIn
                        |
                        +------------------------→ 대상 범위
                   개발자만              모든 분야
```

- **GitHub:** 과정 추적 있지만 개발자 전용 + AI 시대 대응 부족
- **Credly/Accredible:** 범용이지만 "배지 발급"만, 과정 추적 없음
- **LinkedIn:** 가장 넓은 범위지만 자기 신고(self-reported)만, 검증 없음
- **ProveIt:** 과정 추적 깊이 + 범용성의 교차점 = **아무도 없는 자리**

## 8. 확장 로드맵

**Year 1:** 개발자 시장 장악 → "개발자 채용할 때 ProveIt 포트폴리오 있어?"가 당연한 질문이 되게
**Year 2:** 디자이너, 데이터 분석가, 마케터로 확장 → "모든 지식 노동자의 작업 증명"
**Year 3:** 대학 연동 → 학생의 4년간 학습 과정 전체를 포트폴리오로 → 졸업장의 "보완재"에서 "대체재"로
**Year 4:** 글로벌 → 한국(수시 생기부 대안), 인도(DPI 연동), 동남아(학위 위조 문제 심각) 진출
