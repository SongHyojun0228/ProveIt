# marketing.md — ProveIt: 마케팅 전략

---

## 1. 마케팅 핵심 전략: "보여주기 마케팅"

ProveIt의 마케팅은 **제품 자체가 마케팅이 되는 구조**를 설계한다. "우리 제품이 좋다"고 말하는 대신, **사용자의 포트폴리오가 공유될 때마다 ProveIt이 노출**되게 만든다.

핵심 공식:
> 사용자가 포트폴리오 링크를 공유 → 채용 담당자가 본다 → "이거 뭐지?" → ProveIt 인지 → 채용 담당자도 가입 → 더 많은 지원자에게 요구 → 바이럴 루프

이것은 LinkedIn이 초기에 성장한 방식과 동일하다. **양면 네트워크 효과(구직자 ↔ 채용자)**가 핵심이다.

---

## 2. 퍼널 설계

```
[인지] 개발자 커뮤니티에서 ProveIt 발견
  ↓
[관심] "내 코딩 과정을 자동 기록?" → 호기심
  ↓
[시도] 무료로 VS Code 확장 설치, 첫 프로젝트 기록 시작
  ↓
[가치 확인] 1주 후 자동 생성된 프로젝트 요약 확인 → "오, 내가 이렇게 일하는구나"
  ↓
[공유] 포트폴리오 링크를 이력서/LinkedIn에 첨부
  ↓
[바이럴] 채용 담당자가 보고 → 다른 지원자에게도 요구 → 신규 유입
  ↓
[전환] Pro 플랜 (무제한 프로젝트, 상세 인사이트)
  ↓
[확산] 기업이 Enterprise 도입 → 조직 내 표준 도구화
```

**핵심 전환 포인트:** "첫 프로젝트 요약을 본 순간" — 여기서 와우 모멘트가 발생하지 않으면 이탈한다. 따라서 **첫 요약의 품질이 마케팅의 90%**다.

## 3. 채널별 상세 전략

### 3.1 개발자 커뮤니티 (핵심 채널 — 전체 예산의 50%)

**Hacker News / Reddit r/programming**

실행:
- 창업자 본인의 30일 코딩 프로세스 데이터를 공개하는 포스트
- 제목: "I tracked every keystroke for 30 days — here's what I learned about how I actually code with AI"
- 내용: AI 활용 비율, 가장 오래 걸린 버그, 의외의 패턴 발견 등 진짜 인사이트
- ProveIt 언급은 마지막에 자연스럽게 ("I built this tool to do it automatically")
- **절대 광고처럼 보이면 안 됨.** 가치 있는 콘텐츠가 먼저.

목표: HN 프론트페이지 1회 → 초기 사용자 2,000-5,000명

**Disquiet (한국)**

실행:
- "사이드 프로젝트 과정을 ProveIt으로 기록하면서 만든 ProveIt" 메타 스토리
- Disquiet의 "프로덕트 런칭" 기능 활용
- 한국 개발자 커뮤니티에서 초기 300-500명 확보

**velog / 요즘IT**

실행:
- 기술 블로그 시리즈 (3-5편)
  1. "AI 시대, 개발자 포트폴리오가 달라져야 하는 이유"
  2. "VS Code에서 코딩 과정을 자동 추적하는 확장 만들기 (기술 구현기)"
  3. "내 코딩 데이터 30일 분석: AI를 35% 사용하고 있었다"
  4. "채용 담당자에게 물어봤다: 프로세스 포트폴리오가 정말 도움이 되나?"
  5. "ProveIt 오픈 베타 런칭 — 무료로 시작하세요"

**GitHub 오픈소스**

실행:
- VS Code 확장의 코어 로직을 오픈소스로 공개
- "우리는 당신의 데이터를 가져가지 않는다"를 코드로 증명
- 오픈소스 기여자 → 자연스러운 얼리 어답터 + 에반젤리스트

### 3.2 Product Hunt (런칭 이벤트)

**타이밍:** MVP 완성 후, 초기 사용자 500명의 피드백을 반영한 버전으로

**준비물:**
- 60초 데모 영상: "코딩 시작 → 자동 기록 → 1주 후 포트폴리오 생성" 과정
- 스크린샷 5장 (대시보드, AI 활용 비율, 프로젝트 요약, 채용자 뷰, 성장 리포트)
- 첫 번째 코멘트: 창업자가 만든 이유 + 자기 데이터 공개

**목표:** Top 5 of the Day → TechCrunch/The Verge 픽업 가능성

### 3.3 LinkedIn (채용 담당자 타겟)

**오가닉 콘텐츠:**
- 주 2회 포스트 (창업자 개인 계정)
- 주제:
  - "85%의 기업이 스킬 기반 채용을 한다고 말하지만, 실제로는 0.14%만 실행한다. 왜?"
  - "이력서에 'React 3년'이라고 쓰여 있으면, 그걸 어떻게 검증하시나요?"
  - "코딩 테스트 vs 프로세스 포트폴리오 — 어느 쪽이 실제 업무 능력을 더 잘 보여줄까?"
  - "우리 팀에서 ProveIt으로 채용한 첫 케이스 — 면접 시간 40% 단축"
- 형식: 짧은 텍스트 (1,000자 이내) + 데이터 시각화 이미지 1장

**LinkedIn Ads (유료, Phase 2 이후):**
- 타겟: HR Manager, Tech Recruiter, Engineering Manager (한국 + 미국)
- 메시지: "Stop guessing. See how candidates actually work."
- CTA: 무료 Enterprise 파일럿 신청
- 예산: 월 $500-1,000 (초기 테스트)

### 3.4 부트캠프/대학 파트너십 (B2B2C)

**접근 방식:**
- 코드스테이츠, 항해99, 엘리스, 스파르타코딩클럽 등 한국 주요 부트캠프 10곳 리스트업
- 제안: "수료생에게 ProveIt 포트폴리오를 무료 제공 → 취업률 차별화"
- Team 플랜 3개월 무료 파일럿 → 수료생 취업 성공 사례 확보 → 유료 전환

**왜 부트캠프가 받아들이는가:**
- 부트캠프의 KPI는 "취업률"
- 수료증은 모두 가지고 있어서 차별화 안 됨
- ProveIt 포트폴리오가 수료생의 취업에 도움이 되면 → 부트캠프의 마케팅 소재가 됨
- "우리 부트캠프 수료생은 ProveIt 포트폴리오와 함께 졸업합니다" = 강력한 셀링 포인트

**목표:** 3개 부트캠프 파트너십 → 수료생 500명 온보딩 → 취업 성공 사례 10건 확보

### 3.5 SEO / 콘텐츠 마케팅 (장기)

**타겟 키워드:**
- 영문: "developer portfolio 2026", "AI coding portfolio", "process-based portfolio", "prove coding skills", "skills-based hiring tools"
- 한문: "개발자 포트폴리오 만들기", "코딩 테스트 대안", "스킬 기반 채용", "AI 시대 개발자 취업"

**콘텐츠 허브:**
- proveit.dev/blog → 주 1-2회 포스트
- 카테고리:
  - "How I Work" 시리즈: 사용자(동의 하에)의 코딩 패턴 분석 사례
  - "Hiring Better" 시리즈: 채용 담당자를 위한 인사이트
  - "Build in Public" 시리즈: ProveIt 자체의 개발 과정을 ProveIt으로 기록하며 공개

## 4. 그로스 핵: "포트폴리오 워터마크"

**메커니즘:**
- 무료 사용자의 포트폴리오 링크 하단에 "Built with ProveIt — Track your process" + 링크
- 채용 담당자가 지원자의 포트폴리오를 볼 때마다 ProveIt 노출
- Pro 플랜은 워터마크 제거 가능

**이것이 핵심 바이럴 루프:**
```
개발자 A가 무료로 포트폴리오 생성
  → 이력서에 링크 첨부
  → 채용 담당자 B가 열람 → "ProveIt이 뭐지?" → 사이트 방문
  → 채용 담당자 B가 다른 지원자에게도 "ProveIt 포트폴리오 있으면 보내주세요"
  → 개발자 C, D, E가 ProveIt 가입
  → 반복
```

**기대 효과:** 포트폴리오 1개당 평균 3명의 채용 담당자가 조회한다고 가정 → 사용자 1,000명이면 채용 담당자 3,000명 노출 → 이 중 5%가 사이트 방문 = 150명 → 이 중 20%가 다른 지원자에게 요구 = 30명의 "수요 생성자"

## 5. 유료 전환 전략

### 무료 → Pro 전환 트리거

**트리거 1: 프로젝트 제한**
- 무료는 2개 프로젝트까지. 3번째 프로젝트를 시작하면 "Pro로 업그레이드하면 무제한"

**트리거 2: 채용자 조회 알림**
- "누군가 당신의 포트폴리오를 조회했습니다. Pro에서 누가 봤는지 확인하세요."
- (LinkedIn Premium과 동일한 전략 — 검증된 전환 메커니즘)

**트리거 3: 심화 인사이트 잠금**
- 기본 요약은 무료. "AI 활용 패턴 분석", "역량 태그", "성장 리포트"는 Pro 전용
- 무료 사용자에게 인사이트의 일부를 보여주고 나머지를 블러 처리

**트리거 4: 취업 시즌 (3-5월, 9-11월)**
- 취업 시즌에 맞춘 한정 할인: "지금 Pro 시작하면 첫 달 50%"
- "이번 달 포트폴리오 조회 수가 평소의 3배입니다" 알림

### Pro → Enterprise 전환

**접근:** 
- Pro 사용자 중 같은 회사 이메일 도메인이 3명 이상이면 → "팀으로 사용하시나요?" 제안
- Enterprise 무료 파일럿 2주 → 채용 파이프라인에 통합 → 유료 전환

## 7. 예산 배분 (월 기준, Phase 1)

**총 마케팅 예산: 월 200만 원 (부트스트래핑 단계)**

| 항목 | 금액 | 비율 | 용도 |
|------|------|------|------|
| 콘텐츠 제작 | 0원 | 0% | 창업자 직접 작성 (기술 블로그, 데이터 분석) |
| 커뮤니티 활동 | 0원 | 0% | HN, Reddit, Disquiet, velog 직접 활동 |
| 디자인/영상 | 50만 원 | 25% | Product Hunt 런칭 영상, 스크린샷, 소셜 이미지 |
| 부트캠프 제안서 | 30만 원 | 15% | 제안서 디자인, 미팅 비용 |
| LinkedIn Ads | 70만 원 | 35% | 채용 담당자 타겟 (Phase 2부터) |
| 도구/인프라 | 50만 원 | 25% | 서버, Claude API, 분석 도구 |

**Phase 1 (0-3개월) 핵심:** 돈을 쓰지 않는 채널(커뮤니티, 콘텐츠, 오픈소스)에서 최대한 견인. 유료 마케팅은 PMF(제품-시장 적합성) 확인 후 시작.

## 9. 실패 시 피봇 시나리오

**시나리오 1: 개발자가 "감시 느낌"이라서 안 쓴다**
→ 피봇: 개인 대시보드(자기 분석)에 집중. 채용자 공유 기능은 나중에. "나의 코딩 습관 분석기"로 리포지셔닝.

**시나리오 2: 채용 담당자가 "안 봐도 돼"라고 한다**
→ 피봇: B2C 자기계발 도구로 전환. "Strava for Coding" — 운동 기록하듯 코딩 기록. 게이미피케이션.

**시나리오 3: GitHub이 같은 기능을 추가한다**
→ 피봇: GitHub 연동 도구로 전환. GitHub 데이터 + ProveIt 인사이트 = "GitHub 위에 얹는 분석 레이어". 적이 아닌 보완재.
